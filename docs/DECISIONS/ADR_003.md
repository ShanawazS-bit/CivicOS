# ADR-003

Title:
Hackathon Scope — High-Fidelity Illusion, Not Full Platform

Date:
2026-06-25

Status:
Accepted

Problem:
72-hour hackathon cannot deliver custom ML models, full auth ecosystems, or real infrastructure routing.

Decision:
Build a high-fidelity illusion of CivicOS: frictionless ingestion, real-time AI processing, beautiful dashboard. Mock predictive intelligence with hardcoded rules.

Reason:
Aligns with revised `guide.md`. Maximizes demo impact per hour invested.

Alternatives:
- Full platform (profiles, verification, cluster PostGIS queries) — rejected for time

Consequences:
- Single `issues` table is MVP schema; extras deferred
- Cluster detection = frontend rule (3 overlapping pins), not ML
- Voice = browser Web Speech API, not custom STT
- Community verification deferred to post-hackathon
