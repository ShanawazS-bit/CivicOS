# ADR-002

Title:
Supabase + PostGIS for Backend

Date:
2026-06-25

Status:
Accepted

Problem:
Need auth, storage, realtime, and spatial queries in a hackathon timeframe.

Decision:
Use Supabase with PostGIS extension for all backend needs.

Reason:
Single platform for auth, DB, storage, realtime. PostGIS handles spatial deduplication natively.

Alternatives:
- Firebase (weaker spatial support)
- Custom Node backend (too much setup for 72h)

Consequences:
- Vendor dependency on Supabase
- RLS policies required for security
- PostGIS queries via RPC functions
